#PVM 简介

PVM（Polyglot Version Manager） 是一个跨平台的多语言运行时版本管理器，用 Go 编写。 它让你在一台机器上干净地安装、切换、隔离多个语言运行时（runtime）版本， 而不污染系统 PATH、也不需要管理员权限。

#它解决什么问题

场景传统做法的痛点PVM 的做法
多项目用不同 Node 版本全局只能装一个，来回卸载重装pvm use node@18 / node@20 按项目切换
Python 2/3 共存系统依赖冲突独立安装目录，互不影响
Go / Git 全局工具手动管理 PATH统一管理到 ~/.pvm/installs
团队协作版本不一致.nvmrc 各写各的统一 .pvmrc，pvm use 自动生效

#核心特性

  • Shim 驱动：每个可执行文件在 ~/.pvm/shims 下生成轻量转发脚本，PATH 只需指向 shims 目录。
  • 项目级隔离：.pvmrc 声明项目所需版本，pvm use 自动读取并生效，无需管理员权限。
  • 多语言支持：node、python、pnpm、yarn、go、git。
  • 4 级版本解析优先级：环境变量 → 项目 .pvmrc → 用户全局 → 系统 PATH，清晰可预测。
  • 国内镜像：默认走国内镜像下载，可选 --official 回退官方源。

#支持的运行时

运行时作用域说明
node用户级 / 项目级 / 系统级JavaScript 运行时
python用户级 / 项目级 / 系统级Python 解释器
pnpm用户级 / 项目级 / 系统级Node 包管理器（依赖 node）
yarn用户级 / 项目级 / 系统级Node 包管理器（依赖 node）
go仅用户级 / 系统级全局工具，不支持项目级
git仅用户级 / 系统级全局工具，不支持项目级
提示

go 和 git 是"全局唯一"型工具：同一时刻在 ~/.pvm/installs/<rt>/ 下只保留一个版本， 不支持写入项目级 .pvmrc。
