# setup / doctor

## setup

首次初始化 PVM 目录结构并把 shims 加入 PATH。

```bash
pvm setup            # 手动初始化
pvm setup-path       # 仅检查并修复 PATH 配置
```

`setup-path` 会诊断 PATH 是否包含 `~/.pvm/shims`、`~/.pvm/bin`，
并提示与其它版本管理器（如 nvm）的冲突。

### 与 nvm / 系统 node 的冲突处理

如果系统 PATH 中存在 nvm 的 node（如 `D:\node`、`D:\nvm`）或系统 nodejs（`D:\nodejs`），
这些路径会覆盖 pvm 的用户级 shim——因为 Windows PATH 合并顺序为"系统 PATH 在前 → 用户 PATH 在后"。

`setup` 检测到冲突后会自动请求管理员权限，**把 pvm shims 前置到系统 PATH 最前**：

```
系统 PATH = [~/.pvm/shims, ~/.pvm/bin, ..., D:\node, D:\nvm, D:\nodejs]
              ↑ pvm 在最前，压过 nvm
```

这样 pvm 的 shim 优先于 nvm 的软链接，`node` 命令走 pvm 的版本。nvm 的路径保留在末尾（下次重开终端即删除即可）。

### 单二进制

PVM 是单一二进制——不需要单独的 pvm-shim。`reshim` 把 `~/.pvm/bin/pvm` 硬链接为各命令名（`node`、`npm`、`git` 等），
pvm 启动时按自身文件名自分发到对应版本。详见 [Shim 机制](/guide/shim)。

## doctor

检查 PVM 安装健康度。

```bash
pvm doctor
```

检查项包括：

- 根目录 `~/.pvm` 与各子目录是否存在。
- shims 目录是否在 PATH 中（优先级是否够高）。
- 已安装版本的完整性。

:::warning 注意
修改 PATH 后请重启终端 / IDE，否则 `doctor` 可能仍报 PATH 未生效。
:::
