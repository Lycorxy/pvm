# 安全策略

## 报告安全漏洞

如果你发现 pvm 存在安全漏洞，**请不要在公开 Issue 中报告**。

请发送邮件至：**pvm-security@example.com**

我们会在 48 小时内确认收到报告，并在 7 天内给出初步评估和修复计划。

### 报告应包含

- 漏洞的详细描述
- 复现步骤
- 受影响版本
- 如果方便，请提供可能的修复建议

### 处理流程

1. 你报告漏洞
2. 我们确认并评估严重程度
3. 开发修复补丁
4. 发布新版本和安全公告
5. 在公告中致谢报告者（如你希望匿名，请注明）

## 支持版本

| 版本 | 安全支持 |
|------|---------|
| 最新版 | ✅ |
| 旧版本 | ❌ |

请确保使用最新版本。

## pvm 安全设计

- **perUser 安装**：pvm 默认安装到用户目录（`%LocalAppData%\pvm`），不需要管理员权限
- **PATH 管理**：只操作用户级 PATH 注册表，不影响系统级配置
- **自动修复**：检测到系统 PATH 冲突时，通过 UAC 明确请求授权后才修改系统 PATH
- **下载校验**：通过官方源下载运行时，自动验证完整性

## 上报杀软误报

如果 pvm 被防病毒软件误报为恶意文件：

1. 向微软提交：[Windows Defender 文件提交](https://www.microsoft.com/en-us/wdsi/filesubmission)
2. 向你的杀毒软件厂商提交误报
3. 在 Issue 中告知我们（附上杀软名称和版本）

## 已知的安全考虑

- pvm 目前未使用商业代码签名证书，Windows 可能对无签名 exe/msi 发出警告。这是一个可用性问题，而非安全问题。
- 如果你在安全敏感的环境中使用，建议从源码编译：`go build -o pvm .`
